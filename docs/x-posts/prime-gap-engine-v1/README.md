# Prime Gap Generative Engine v1.0

We reduced each prime gap to a small deterministic type: first open offset,
selected-integer family, and a coarse divisor bucket. On that reduced surface, the gap
stream does not look memoryless. It closes to a stable `14`-state grammar that
persists from sampled `10^7` through `10^18`.

![Prime Gap Generative Engine v1.0 overview](../../../output/gwr_dni_gap_type_engine_v1_overview.png)

---

Three states dominate the dynamics:

- `o2_odd_semiprime|d<=4`
- `o4_odd_semiprime|d<=4`
- `o6_odd_semiprime|d<=4`

We call that triad the **Semiprime Wheel Attractor**.

From outside the triad, the next state returns to it about two thirds of the
time. Once inside it, the walk stays inside it about two thirds of the time.

---

The first important result is generative, not just descriptive. A deterministic
second-order rotor trained below `10^18` beats weaker memory baselines on the
held-out `10^18` window.

- triad-share error: `0.0234`
- reduced-state pair L1: `0.4627`
- pair-top-share error: `0.0078`

That is already enough to say the reduced grammar carries real predictive
structure.

---

The second result is the scheduler. A lag-2 scheduler plus a small periodic
periodic correction closes the local `256`-window surface to pooled L1 `0.0116`.

That is not a vague match. It reproduces the pooled concentration triple

- real: `0.3284 / 0.5059 / 0.7594`
- engine: `0.3269 / 0.5057 / 0.7692`

---

The third result is the long-horizon reset layer. Higher-divisor-triggered
event locks sharpen the stationary walk:

- event-lock `L = 3`: pooled L1 `0.0150`, full-walk three-step `0.5564`
- event-lock `L = 6`: full-walk three-step `0.6278`

So the right picture is not one flat chain. It is a hierarchy:

1. a `14`-state core grammar
2. a scheduler layer
3. a slower reset controller

---

The honest part matters too. No single tested deterministic controller yet
hits both of these at once:

- pooled-window concentration L1 below `0.015`
- full-walk three-step concentration above `0.62`

That tradeoff is not an embarrassment. It is evidence that the engine really
has multiple timescales.
